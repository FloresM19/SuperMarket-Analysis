# Proyecto Practico de Analisis de Datos: SuperMarket Analysis

## Proyecto
Este proyecto consiste en un análisis de datos utilizando un dataset de un supermercado. El objetivo de este es: revisión y limpieza de datos, consultas sql, estadisticas (promedio, media y desviación) y Dashboard. 

# Herramientas
- SQL: PostgreSQL
- Excel 
- Power BI
- Python 
- Word 
- Github

## Dataset 
Invoice ID: Identificador único para cada transacción.
Branch: La ubicación de la sucursal.
City: La ciudad en la que se encuentra la sucursal.
Customer Type: Indica si el cliente es 'Miembro' o 'Normal'.
Gender: Género del cliente.
Product_Line: La categoría del producto vendido.
Unit_Price: Precio por unidad del producto.
Quantity: Número de artículos comprados.
Tax 5%: Monto del impuesto calculado.
Total: Monto total de la transacción incluyendo el impuesto.
Date: Fecha de la transacción.
Time: Hora de la transacción.
Payment: Método de pago utilizado (por ejemplo, Efectivo, Billetera electrónica, Tarjeta de crédito).
COGS: Costo de los bienes vendidos, que representa el costo base de los productos.
Gross_Margin_Percentage: Porcentaje fijo de ganancia para cada venta (4.7619%).
Gross_Income: Ganancia obtenida de la transacción.
Rating: Calificación de satisfacción del cliente (sobre 10).
